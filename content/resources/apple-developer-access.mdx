Security groups for governing external team access to App Store Connect, managed by the MCoE team. All groups are children of the parent group `mcoe_developer_program`.

## Requesting Access

Request group membership through [Secure](https://secure.uhc.com/). Search for the group name that corresponds to your organization and submit an access request.

<Frame>
  ![platform-access-secure](/images/docs/resources/platform-access-secure.png)
</Frame>

## App Store Connect Groups

Assign each `mcoe_asc_*` group to the appropriate App Store Connect role. Assignments are made at the ASC user level to email identities that correspond to group membership.

| Group Name | Description | Parent |
|---|---|---|
| `mcoe_asc_uhc` | Access for UNITED HEALTHCARE SERVICES, INC. | `mcoe_developer_program` |
| `mcoe_asc_optum` | Access for Optum Inc | `mcoe_developer_program` |
| `mcoe_asc_soultran` | Access for Soultran | `mcoe_developer_program` |
| `mcoe_asc_uhg` | Access for UnitedHealth Group | `mcoe_developer_program` |
| `mcoe_asc_uhg_enterprise` | Access for UHG Enterprise | `mcoe_developer_program` |

## ASC Role Reference

| Role | What It Allows |
|---|---|
| **App Manager** | Manage app metadata, pricing, availability, and TestFlight |
| **Developer** | Upload builds, manage TestFlight, access Xcode Cloud |
| **Access to Certificates, Identifiers & Profiles** | Manage signing certificates and provisioning profiles |

<Note>Assign the least-privilege role required. Most app teams only need Developer. App Manager is reserved for teams managing App Store metadata and submissions.</Note>
